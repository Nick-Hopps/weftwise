# Ask AI 召唤式悬浮面板设计

## 背景

当前 Ask AI 固定在右侧上下文面板中。聊天能力完整，但入口依赖 Header 图标，面板长期占用横向空间，并与 Context 标签竞争注意力。

## 目标

- 桌面端允许在正文空白处双击召唤 Ask AI。
- 正文存在文本选区时，面板在选区末尾附近出现并自动携带引用。
- 桌面面板可自由拖动，且仅由关闭按钮或 Escape 主动关闭。
- Header 保留 Ask AI 图标与快捷键入口。
- 移动端使用底部 Sheet，并支持下滑关闭。
- 保留现有会话、SSE、引用、审批和保存到 Wiki 能力。

## 约束

- 不拦截链接、按钮、表单、编辑器、代码块和可编辑区域的双击。
- 不以全局双击替代显式入口，避免降低功能可发现性。
- 弹层必须限制在可视区域内，窗口变化后仍可操作。
- 移动端不提供自由拖动位置，只允许通过顶部手柄下滑关闭。
- 选区引用只在唤起时捕获，面板打开后不跟随选区移动。

## 视觉主张

Ask AI 是贴近阅读位置的轻量工作面，不是新的固定导航栏；使用平静表面、单一强调色和克制阴影，与正文保持清晰层级。

## 内容结构

1. 顶栏：拖动手柄、Ask AI 标识、当前上下文、关闭按钮。
2. 会话栏：沿用会话切换与新建能力。
3. 消息区：沿用流式回答、引用来源和审批卡片。
4. 输入区：沿用选区引用、页面段落引用、停止生成与发送能力。

## 交互主张

- 桌面从触发点附近快速缩放淡入，强调“就地召唤”。
- 拖动使用直接跟手位移，释放后自动约束到安全区域。
- 移动端 Sheet 从底部进入；向下拖动超过距离或速度阈值后关闭，否则回弹。

## 桌面行为

- `main` 内的正文空白区域双击时，以指针坐标为锚点打开面板。
- 若当前存在属于正文的非折叠选区，以选区末端矩形为锚点，并把文本写入聊天引用信箱。
- 面板默认宽 440px、高度不超过 `min(680px, calc(100vh - 64px))`，为拖动保留可感知的安全空间。
- 面板仅能从顶栏拖动，位置在视口内保留至少 16px 安全边距。
- Header 图标和 `⌘J` 从上次位置或右上安全位置打开面板。
- 点击外部不关闭；关闭按钮与 Escape 关闭。

## 移动端行为

- Header 图标与选区 Ask AI 打开底部 Sheet。
- Sheet 高度占视口约 88%，顶部圆角并保留安全区。
- 顶部手柄向下拖动超过 96px 或释放速度达到阈值时关闭，否则回弹。
- 点击遮罩不关闭，避免误丢失对话；关闭按钮与下滑手势负责关闭。

## 成功标准

- 双击正文空白能稳定打开面板，交互控件双击不触发。
- 选中文本唤起后引用完整、面板不遮挡选区末端。
- 面板拖动后不会移出视口，窗口缩放后仍可关闭。
- 移动端 Sheet 可下滑关闭，未达到阈值时回弹。
- 关闭并重新打开不会丢失当前会话或流式请求状态。
- 现有 Ask AI 相关测试通过，并新增定位、触发过滤和下滑判定的纯逻辑测试。
