# 用户界面 Emoji 图标替换设计

## 背景

当前用户界面存在两类由 emoji 承担图标职责的内容：

1. Ask AI 工具活动与 worker 任务事件由 `src/lib/tool-activity.ts` 直接拼接 emoji，因而同时出现在聊天活动列表、任务进度摘要和任务详情日志中。
2. Wiki 正文的 callout 标题会保留 LLM 写入 Markdown 的前导 emoji，历史页面与新生成页面的视觉风格均依赖操作系统 emoji 字体。

不同平台的 emoji 字形、颜色和基线不一致，也无法稳定继承应用的前景色、暗色主题与尺寸规范，削弱了界面整体感。

## 目标

- 面向用户的工具活动、任务活动与正文 callout 不再使用 emoji 充当图标。
- 统一复用项目已有的 `lucide-react` 线性图标集。
- 同一语义在 Ask AI、任务面板与任务详情中使用同一图标。
- 历史 Markdown 无需迁移或改写，渲染时即可获得新图标样式。
- 服务端与共享纯逻辑不依赖 React 或 `lucide-react`。
- 图标保持可访问性：装饰图标不进入读屏文本，现有可读标签保持不变。

## 非目标

- 不移除用户正文、标题、标签等自由内容中主动输入的 emoji。
- 不批量改写 vault 中的 Markdown。
- 不重新设计 Ask AI、任务面板或 callout 的整体布局。
- 不引入第二套图标包，也不新增彩色插画式图标。

## 视觉命题

在现有低噪声知识工作台中，以 14–16px、单色、1.5–2px 视觉笔画的 Lucide 图标建立稳定的语义节奏；颜色只表达状态或 callout 分类，不增加图标底座、阴影或额外卡片。

## 方案比较

### 方案 A：语义键 + React 图标组件映射（推荐）

共享纯逻辑只返回稳定的语义键或工具名，客户端组件集中把语义映射为 Lucide 组件。worker 日志仅写纯文本，并在事件 data 中保留工具名。

- 优点：服务端边界清晰；图标跨界面一致；易测试；不污染日志与复制文本。
- 代价：需要新增一个很薄的客户端图标适配组件，并让任务表面从事件 data 读取工具类型。

### 方案 B：CSS mask / data attribute

共享逻辑输出字符串类名，CSS 使用 SVG mask 绘制图标。

- 优点：DOM 简单。
- 代价：重复维护 Lucide 路径，类型约束弱，主题与构建调试成本更高。

### 方案 C：迁移现有 Markdown 与事件文案

批量删除 vault 中的 emoji，并让 prompt 生成其他标记。

- 优点：源文本整洁。
- 代价：会产生大规模知识内容变更，无法修复既有 job 历史，回滚与审计成本不符合本次视觉需求。

采用方案 A。

## 详细设计

### 工具活动

- `toolActivityIcon` 不再返回 emoji；改为返回受控的 `ToolActivityIconName` 语义键。
- `toolActivityLine` 只生成可复制、可检索的纯文本文案，不再把视觉图标写入持久化事件消息。
- 新增 UI 适配组件，根据工具名或语义键渲染 Lucide 图标。
- Ask AI 活动列表直接渲染该组件。
- 任务面板、单任务进度与详情日志从最新/逐条事件的 `data.tool` 读取工具名；仅对工具事件展示工具图标，普通阶段继续使用现有状态图标或无图标日志。
- 对历史事件消息中的已知前导 emoji 做展示层清理，避免旧任务日志残留彩色字形。

### Wiki callout

- callout remark 转换继续以 `[!type]` 作为唯一类型来源。
- 对受支持 callout 的标题首行，只剥离“行首已知 emoji + 空白”，保留用户写入的实际标题文字；正文其他位置的 emoji 原样保留。
- 为 callout 输出稳定的自定义图标节点，并在 `rehype-react` 中映射为 Lucide 图标组件。
- 支持 `intuition`、`example`、`quiz`、`background`、`diagram`、`pitfall`，未知类型使用通用 `Info` 图标。
- 图标与首段标题同行；无显式标题时仍显示类型 eyebrow 与图标，但不制造空标题文案。

### 其他固定状态文案

- 重置成功/失败等程序生成的聊天文案移除前导 emoji；状态意义继续由明确文字承担，避免把自由 Markdown 消息误判为结构化状态组件。
- prompt 中要求生成的固定警告 emoji 改为标准 callout 语法，由 callout 图标系统负责视觉表达。

## 成功标准

- `src` 的非测试运行时代码中，不再存在由产品固定写入的用户界面 emoji。
- Ask AI 工具活动显示对应 Lucide 图标，图标不参与读屏。
- Fix/Curate 等任务的当前工具调用与详情日志不再显示 emoji，并能显示同语义 Lucide 图标。
- 历史 `> [!intuition] 💡 直觉` 渲染为 Lucide 图标 + “直觉”，原始 Markdown 不变。
- 普通正文中的用户 emoji 不被剥离。
- 亮色、暗色与窄屏下图标尺寸、基线、对比度稳定。
