# 正文固定目录设计

## 背景

Wiki 阅读页已经提供阅读进度、正文选区 Ask AI、来源分栏和右侧 Context 检查器，但长正文缺少章节级导航。用户滚动离开页首后无法快速了解当前位置，也需要依赖浏览器查找或大幅滚动才能跳到其他章节。

## 目标

- 从正文 Markdown 的二至四级标题生成目录，目录文本与正文标题锚点使用同一解析结果。
- 在阅读过程中持续提供目录入口，并突出当前章节。
- 点击目录项后在正确的滚动容器内平滑跳转，普通阅读与 Sources 分栏模式都可用。
- 宽内容区常驻显示目录；空间不足时收敛为粘性紧凑入口，不挤压正文阅读宽度。
- 正文切换为 Reshape 结果时，目录和锚点同步更新。

## 非目标

- 不把目录写回 Markdown 或数据库。
- 不改变现有跨页面 wikilink 的 `#Section` 解析语义。
- 不为编辑器预览、聊天消息或来源文档增加常驻目录。
- 不提供目录层级折叠、拖拽排序或用户自定义显隐设置。

## 现状约束

- 普通阅读由 `#main-content` 滚动；Sources 分栏下左侧正文容器独立滚动。
- Context 面板可关闭且用于 backlinks、frontmatter 和 mini graph，不适合作为正文目录的唯一入口。
- `renderMarkdown()` 被聊天、来源预览等多个表面复用，不能默认给所有标题注入全局锚点。
- 页面首个一级标题会被隐藏，页头标题由 frontmatter 单独渲染，因此目录不应收录一级标题。

## 方案比较

### 方案 A：扩展右侧 Context 面板

在 Context 中增加 Outline 区域或 Tab。

优点：改动集中，天然固定。缺点：Context 可关闭，移动端是临时抽屉；打开 Sources 或窄屏时入口不持续可见，也会把页面关系检查和正文导航混在一起。

### 方案 B：正文右侧永久固定栏

无论可用宽度都为目录预留固定宽度。

优点：形态稳定、实现直接。缺点：主区域同时受左侧栏、Context 和 Sources 分栏影响，强制保留目录会明显压缩正文，窄屏不可用。

### 方案 C：正文内自适应双形态目录（推荐）

以主内容区的实际容器宽度为准：足够宽时使用正文右侧粘性目录轨道；空间不足时使用正文顶部粘性紧凑入口，展开为浮层目录。两种形态复用同一目录数据、当前章节和跳转函数。

优点：目录始终可达，不依赖 Context 开关；不牺牲窄内容区的正文宽度；适配 Sources 分栏。代价：需要维护两种视觉形态，并正确识别两类滚动容器。

## 推荐设计

### 标题与锚点

新增纯函数解析 Markdown AST，只收集 `h2`、`h3`、`h4`：

- 递归提取标题节点中的可见文本，包括强调、链接、行内代码等子节点。
- 将文本规范为小写、以连字符连接的稳定 ID，保留 Unicode 字母与数字。
- 重复标题依次追加 `-2`、`-3`；空标题使用 `section` 兜底。
- 目录提取和渲染时的标题 ID 注入共用同一套遍历与去重规则。

`renderMarkdown()` 新增显式 `headingAnchors` 选项，仅 `PageRenderer` 开启，避免聊天和来源预览产生重复 DOM ID。

### 展示形态

- 当正文少于两个可导航标题时不展示目录，避免为短页面增加噪声。
- 宽内容区：正文保持既有最大阅读宽度，右侧新增约 220px 目录轨道；目录使用 `position: sticky`，顶部与阅读进度留出安全距离。
- 窄内容区：显示带 List 图标和当前章节名的粘性按钮；点击展开目录浮层，再次点击、选择章节、点击外部或按 Escape 后关闭。
- 目录项按标题深度缩进，最长文本截断并通过 `title` 保留完整内容。
- 当前章节使用强调色左边界和前景色区分，其余项保持低对比度。

### 当前章节与跳转

- 监听实际滚动容器的 scroll 与 resize，读取当前页面全部目录标题的 `getBoundingClientRect()`。
- 以滚动容器顶部以下固定偏移为判定线；最后一个越过判定线的标题为当前章节，页首时选择第一个章节。
- 点击目录项调用目标节点 `scrollIntoView({ behavior: 'smooth', block: 'start' })`，并同步更新 URL hash，保留浏览器可分享和前进后退语义。
- 标题设置 `scroll-margin-top`，避免粘性紧凑目录遮挡目标标题。

### 可访问性

- 两种目录都使用 `<nav aria-label="Table of contents">`。
- 当前项使用 `aria-current="location"`。
- 紧凑入口使用 `aria-expanded`、`aria-controls`；浮层支持 Escape 和外部点击关闭。
- 用户开启 reduced motion 时使用即时滚动。

## 成功标准

- 含重复、中文、格式化标题的正文生成唯一且可跳转的稳定目录。
- 普通阅读和 Sources 分栏都能高亮当前章节并跳转。
- 宽主内容区目录常驻；窄主内容区目录入口粘性可达，正文宽度不被永久压缩。
- Reshape 原文/重塑结果切换后不保留旧目录项。
- 页面无标题或只有一个二至四级标题时不显示目录。
